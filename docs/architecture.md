## 模块分层说明

### 前端层
- **Mobile Web UI**: 基于 HTML/CSS/JS 的移动优先界面
- **REST API Client**: 外部程序通过 HTTP 调用引擎服务
- **Third-party Plugin**: 第三方插件通过 API 集成

### API 层
- **FastAPI**: 现代高性能 Python Web 框架
- **Uvicorn**: ASGI 服务器，端口 8897
- 端点: 聊天、引擎、会话管理、治理、管线、合成、智能体

### 引擎层
- **PipelineOrchestrator**: 引擎管线编排器，负责管理多个引擎的执行顺序
- **SynthesisEngine**: 多引擎融合合成器
- **AgentEngine**: 自主任务执行智能体
- **引擎模块矩阵**: 12 个专业引擎模块

### 核心层
- **ConfigLoader**: 多源配置管理（YAML/JSON/ENV）
- **RateLimiter**: 令牌桶 + 滑动窗口限流
- **Cache**: 内存 + 磁盘双层缓存
- **LoggingSetup**: 分级日志系统

### 数据层
- **SQLite**: 持久化存储（会话、治理日志）
- **VectorStore**: TF-IDF 向量存储用于 RAG
- **Sessions**: 内存中的会话状态
- **KnowledgeBase**: 内存中的知识库

## 扩展点说明

### 插件系统

```
+-------------------------------+
|        Plugin Manager          |
|                                |
|  +----------+  +----------+    |
|  | Plugin A |  | Plugin B |    |
|  | (skill)  |  | (tool)   |    |
|  +----------+  +----------+    |
|          |            |         |
|  +-------v------------v------+ |
|  |      Plugin Loader         | |
|  |  - 动态加载 .py 插件       | |
|  |  - 热插拔机制              | |
|  |  - 依赖检查                | |
|  +----------------------------+ |
+-------------------------------+
```

插件类型:
1. **技能插件 (Skill)**: 扩展引擎能力（如代码解释器、翻译器）
2. **工具插件 (Tool)**: 提供外部工具集成（如天气查询、股票信息）
3. **中间件插件**: API 层面的拦截和增强
4. **数据源插件**: 新增知识库/数据源连接器

注册方式:
```python
from cee.app.plugins.loader import register_plugin

@register_plugin(name="my_plugin", version="1.0", plugin_type="skill")
class MyPlugin:
    def execute(self, input_data: dict) -> dict:
        ...
```

### 引擎模块注册

新增引擎模块通过 `__init__.py` 导出自动注册:

```python
# src/cee/app/engine/new_engine.py
class NewEngine:
    def process(self, input_text: str) -> str:
        ...

# src/cee/app/engine/__init__.py
from cee.app.engine.new_engine import NewEngine
```

### 管线扩展

预定义管线可通过 YAML 配置扩展:

```yaml
# custom_pipelines.yaml
pipelines:
  my_custom_pipeline:
    steps:
      - name: search
        depends_on: []
      - name: think
        depends_on: [search]
      - name: summarizer
        depends_on: [think]
```
